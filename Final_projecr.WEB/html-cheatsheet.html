<html>

The <html> element is the root container for an HTML document. Everything you want the browser to render—headings, paragraphs, images, scripts—must be nested inside this element. It declares the start and end of an HTML page and globally defines the document’s language and metadata scope.

<head>

The <head> element holds metadata—like the page title, character encoding, stylesheet links, and scripts—that doesn’t display directly on the page. Browsers use this information to properly render and index your site, making it critical for performance, SEO, and linking external resources.

<title>

Placed inside the <head>, the <title> element sets the text shown in the browser tab and search engine results. It should be concise yet descriptive, helping users and crawlers understand the page’s main topic at a glance for usability and SEO.

<body>

The <body> element wraps all visible webpage content—headings, paragraphs, images, links, and user interface elements. It defines the main document structure that browsers render. Anything you want the visitor to see or interact with must live inside this container.

<h1>–<h6>

Heading tags (<h1> through <h6>) define section titles in descending importance. Search engines and screen readers use them to understand page structure. Use one <h1> per page for the main title, and nest smaller headings logically for subsections.

<p>

The paragraph tag (<p>) defines a block of text, automatically adding space before and after for readability. It’s ideal for sentences and short blocks of content. Don’t use it for layout—keep it strictly for meaningful text.

<a> (href, target)

The anchor tag (<a>) creates hyperlinks. The href attribute specifies the destination URL. Use target="_blank" to open links in a new tab, and rel="noopener" for security. Anchors allow navigation within and between sites.

<img> (src, alt)

The image tag (<img>) embeds pictures. Use the src attribute for the file path and alt text to describe images for accessibility and SEO. Always provide meaningful alt text to support non-visual browsing and improve indexing.

<ul>, <ol>, <li>

Unordered (<ul>) and ordered (<ol>) lists organize items. Each item goes in an <li>. Use lists for navigation menus, steps, and bullet points. They improve scannability and semantics compared to manually styled paragraphs.

<table>, <tr>, <td>, <th>

Table elements structure tabular data. <table> contains rows (<tr>), which hold cells: data cells (<td>) and header cells (<th>). Use captions and summaries for accessibility and clear data presentation.

<form> (action, method)

The <form> tag wraps input controls to submit data. action defines the server endpoint, and method specifies GET or POST. Forms power login pages, surveys, and contact forms—fundamental for interactive sites.

<input> (type, placeholder)

The <input> tag creates form controls—text fields, passwords, radios, checkboxes. The type attribute defines control behavior, and placeholder offers hint text inside empty inputs to guide users on required format or content.

<button>

Buttons (<button>) submit or reset forms (type="submit" / type="reset") or trigger JavaScript (type="button"). Unlike <input>, they can contain HTML content, making them more flexible for styled interactive controls.

<section>, <article>, <aside>

Semantic tags improve document structure: <section> groups related content, <article> encapsulates self-contained content (e.g., blog post), and <aside> holds tangential content like sidebars. They boost accessibility and SEO.

Attributes: class, id, style

class groups elements for CSS/JS targeting; id uniquely identifies one element; style applies inline CSS rules directly. Use classes for reusable styles, IDs for single elements, and inline only sparingly.
